█ ZEN-TRACKER (v1.2.0)
================================

A minimal, TUI-driven focus engine for purists.

Zen Tracker is not another polished productivity app trying to sell me a subscription or wrap itself in pastel nonsense. I built it because I wanted a focus tool that looked like it belonged in a terminal scratchpad. No cloud, no tracking, no bloat. Just tasks, a timer, and a shell-ish interface.

🛠 Why I Built This
-------------------

I got tired of web timers that felt too soft and too generic. I live in the terminal most of the day, and I wanted something that felt like a native control panel instead of a random Chrome tab pretending to be useful.

The UI is intentionally sharp, monospace, and high-contrast. It sits nicely in a side-tiled window while I’m coding, without looking like it was generated by a marketing team.

🚀 What It Actually Does
------------------------

- Pomodoro + Custom Sessions: run strict sessions and set durations from 1 to 180 minutes.
- Persistent Timer State: if you refresh mid-session, the countdown resumes from saved state.
- Task Queue + Priority Tags: enqueue tasks with [LOW], [MEDIUM], [HIGH] labels.
- Local Persistence: timer, tasks, theme, audio mode, and preferences stay saved in-browser.
- In-App Notifications: completion and control feedback in the UI, not just blocking system dialogs.
- Optional Browser Notifications: can be enabled and permission-gated from the app.
- Ambient Focus Audio: choose between `lofi`, `rain`, `fire`, and `forest` while tracking.
- Theme Customization: switch between `violet`, `ocean`, `ember`, `forest`, and `mono` palettes.
- Streaks + History: tracks focus minutes, sessions today, and streak metrics from completed sessions.
- Terminal Commands: command bar supports `help`, `run`, `pause`, `reset`, `set`, and `add`.
- Custom Cursor Effect: animated cursor trail on supported desktop pointers.

🧱 Current State
---------------

- Terminal-inspired layout with configurable theme presets.
- Local-only data model (no backend, no accounts, no sync).
- Runtime audio synthesis for ambience and alarm (no external audio assets required).
- Browser support can vary for autoplay policy and notification permissions.

🪦 The Struggles / Bugs That Almost Killed Me
--------------------------------------------

- I broke the main JS file once while refactoring event listeners and spent way too long chasing one missing brace.
- Swapping between rounded cards and harder terminal edges caused a bunch of style cleanup.
- I have definitely run a command in the wrong directory and confused myself for no good reason.
- Wayland/Hyprland setups can get weird fast, so I keep expecting some tiny UI detail to behave differently on another machine.

📦 Installation
--------------

This is a static web app, so there is nothing to install.

Option A
- Open `index.html` directly in your browser.

Option B
- Run a local server:

```bash
python3 -m http.server 8000
```

- Then open:

```text
http://localhost:8000
```

⌨️ Commands
----------

- `help` - list available terminal commands.
- `run` / `start` - begin the timer.
- `pause` / `stop` - pause the timer.
- `reset` - reset the timer back to 25:00.
- `set <minutes>` - set session duration (for example: `set 45`).
- `add <task> [high|medium|low]` - queue a task from the terminal prompt.
